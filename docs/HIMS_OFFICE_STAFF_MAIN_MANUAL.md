# HIMS Office Staff Main Manual

## 1. Purpose
Provide day-to-day guidance for Hanmarine office staff responsible for crew management, compliance, and finance using the HIMS platform and supporting processes.

## 2. Roles and Responsibilities
- **HR Staff**: maintain crew records, manage applications, track training.
- **Operations Staff**: schedule assignments, coordinate embarkation logistics.
- **Compliance Staff**: monitor certificate validity, update external system statuses.
- **Accounting Staff**: prepare payroll, manage allotments, reconcile invoices.
- **Directors**: approve critical transactions, review KPIs, audit trails.

## 3. Daily Routine Checklist
1. Login to HIMS by 08:30 local time.
2. Review Dashboard alerts (compliance, assignments, finance widgets).
3. Clear notifications requiring same-day action.
4. Update crew changes (applications, assignments) before 12:00.
5. Afternoon review of pending approvals and export required reports.
6. Log out or lock workstation when leaving desk.

## 4. Key Processes

### 4.1 New Crew Onboarding
1. HR creates application in Applications module.
2. Upload documents: passport, medical, certificates.
3. Conduct screening, interview, and background checks.
4. Approve and convert to Crew record; notify Operations.
5. Create SEA contract draft; route to Director for approval.

### 4.2 Assignment Planning
1. Operations checks upcoming sign-on roster.
2. Confirm vessel readiness with Principal via email.
3. Assign crew in Assignments module; set sign-on/off dates.
4. Generate travel checklist (visa, tickets, clearance).
5. Update status after embark/disembark.

### 4.3 Compliance Monitoring
1. Compliance staff reviews expiring certifications daily.
2. Renew certificates via relevant authority (KOSMA, Dephub, Schengen).
3. Update HIMS compliance records with validity dates and attachments.
4. Flag high-risk cases to Director if renewal delayed >7 days.

### 4.4 Payroll Cycle
1. Accounting imports crew timesheets by the 25th.
2. Validate allowance and deductions with signed forms.
3. Generate payroll batch in Accounting module.
4. Obtain Director approval; export bank file.
5. Archive signed disbursement summary in Documents repository.

### 4.5 Incident Reporting
1. Any incident onboard or at agency logged in Crew > Notes.
2. Create Incident Report form (per SOP) and upload attachments.
3. Notify Director and relevant departments through HIMS comments.
4. Track action items until closure.

## 5. Communication Guidelines
- Use HIMS Comments instead of personal messaging for traceability.
- Tag colleagues with @username for follow-up tasks.
- For urgent matters, escalate via Teams call but document outcome in HIMS notes.
- Maintain professional language; avoid sensitive data in chat channels.

## 6. Documentation Standards
- File names: use YYYYMMDD_CrewName_DocumentType.pdf.
- Store signed contracts in Contracts module; do not keep on desktop.
- Scan documents at 300 DPI; ensure legible text and signatures.
- Retain original hardcopies per compliance policy (min. 5 years).

## 7. Security Practices
- Use company-issued hardware with endpoint protection.
- Never share login credentials; password changes every 90 days.
- Lock workstation when away; screen auto-lock set to 5 minutes.
- Sensitive data (RED level) may only be emailed via secure channel.

## 8. Business Continuity
- In case HIMS unavailable, activate offline forms stored in shared drive.
- Log critical transactions manually and upload to HIMS once restored.
- Contact IT hotline immediately for system downtime >10 minutes.

## 9. Training and Support
- New staff attend onboarding session (2 hours) covering modules.
- Monthly refresher: first Friday, 15:00, led by CDMO.
- Knowledge base located in SharePoint: HIMS > SOP Library.
- IT support: itsupport@hanmarine.co, extension 230.

## 10. Audit Readiness
- Maintain complete records; avoid deleting historical data.
- All approvals must occur in HIMS to ensure audit trail.
- Quarterly internal audit run by Director; staff must cooperate.
- Prepare for authority inspections by exporting compliance reports on request.
